2026-06-30 — Sessão 3: Parte XI — Negociação, contratos e pagamentos
------------------------------------------------------------------

DECISÃO IMPORTANTE (segurança + escopo):
- Pagamento é ESCROW SIMULADO/registrado. A plataforma NÃO movimenta dinheiro real
  (camada futura/opcional do escopo). É uma máquina de estados auditável, sem
  integração financeira real.

BACKEND (módulo "negocios", com auditoria):
- Modelos: Contrato (escopo, valorTotal, prazoDias, status, assinaturas das 2 partes),
  Marco (entregável: titulo/valor/ordem/status), Pagamento (escrow simulado).
- Ciclo de vida: formalizar (de proposta ACEITA) -> RASCUNHO -> assinar (as 2 partes)
  -> ATIVO (cria escrow RETIDO por marco) -> prestador ENTREGA marco -> cliente APROVA
  (escrow RETIDO->LIBERADO, marco PAGO) -> quando todos pagos -> CONCLUIDO.
- Endpoints: GET /contratos, GET/POST /contratos[/:id], POST :id/assinar,
  POST :id/marcos/:marcoId/entregar, POST :id/marcos/:marcoId/aprovar.
- Regras: 1 contrato por proposta (unique), só partes acessam, papéis corretos
  para entregar/aprovar, contrato só ativa com as 2 assinaturas.

FRONTEND:
- Página /contratos (lista + gestão). Componentes ContratoView (assinar/entregar/
  aprovar/escrow) e FormalizarContrato (escopo + marcos dinâmicos).
- Integrado ao detalhe da solicitação: proposta ACEITA mostra "Formalizar contrato"
  ou o contrato existente. Link "Contratos" no menu.
